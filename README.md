# 720Yun / 720VrYun 全自动全景图下载合成工具

这是一个基于 Python Tkinter 的可视化图形界面（GUI）工具，用于全自动解析 720Yun 和 720VrYun 网页链接、下载切片图块并缝合还原为高质量的 2:1 等距柱状全景图。

## 功能特性

1. **自动网页场景解析**：输入网页链接即可一键拉取作品下所有子场景，无需手动分析网络请求寻找直链前缀。
2. **多分辨率图块网格自适应探测**：系统自动探测当前选定清晰度（`l1` ~ `l4`）下的网格行数与列数（如 5x5、3x3 等），跳过复杂的行列参数配置。
3. **扁平化结构输出与临时文件清理**：最终大图直接输出至选定的保存文件夹根目录，任务完成后自动物理删除中途下载的几百个琐碎临时分片文件夹。
4. **防误触 UI 锁定**：【开始下载合成】按钮启动时默认置灰禁用，只有在网页场景解析成功后才被激活。一旦输入框 URL 文本发生改变，按钮将立刻二次置灰重新加锁。
5. **本地授权码缓存校验**：
   - 授权码生成算法与系统日期和小时绑定，万能授权码格式为：`jerry + YYYYMMDDHH` (例如 `2026年7月10日16点` 对应的密码为 `jerry2026071016`)。
   - 输入一次正确授权码后，将自动在本地保存 `.auth_cache.json` 授权缓存。
   - 在有效期（1天）内且抓取次数（5次）未超限时，点击下载直接放行；超出后将自动弹出对话框要求重新输入。操作栏常驻绿/红色高亮标签实时展示当前剩余可用次数。

## 项目打包指南

为了在没有 Python 环境的 Windows 电脑上直接双击使用该工具，我们使用 `PyInstaller` 将其编译打包成独立的 EXE 可执行文件。

### 打包命令

在项目根目录下打开命令行终端（推荐 PowerShell），运行以下打包命令：

```powershell
pyinstaller --onefile --noconsole --name="全景下载合成工具" gui.py
```

### 参数详解

* `--onefile`：将所有 Python 脚本、依赖项、图像算法科学计算库（NumPy、OpenCV、SciPy、Pillow 等）全部压缩打包进一个单一的 `.exe` 运行程序中，方便单文件分发。
* `--noconsole`：运行 EXE 时不弹出命令行窗口（黑框），获得与原生 Windows 桌面软件完全一致的无缝图形化体验。
* `--name="全景下载合成工具"`：指定生成的可执行文件文件名称。

### 打包结果

编译完成后，会在当前项目路径下生成以下两个文件夹：
1. `build/`：打包时的临时工作目录。
2. `dist/`：**打包产物目录**。您最终需要的 `全景下载合成工具.exe` 就在该文件夹中。

## 一键自动打包与双分支 GitHub Release 发布

我们编写了 `release.sh` 自动化 Shell 脚本，帮助您在本地完成打包后，一键自动推送版本并上传 EXE 到 GitHub Release（在此过程中，主发布分支不包含任何源码）。

### 双分支发布规则

* **`master` 分支（开发分支）**：用于您本地开发，提交并保存所有代码（包含 `gui.py`、`util.py` 源码）。
* **`main` 分支（公开/下载分支）**：为发布分支。在此分支下运行的 `.gitignore` 会被重新自动配置为忽略 `gui.py` 和 `util.py` 等源码，因此只公开 `README.md`、`requirements.txt` 和已编译好的打包大图 EXE！

### 使用前提

1. **终端支持**：
   在 Windows 下建议在 **Git Bash** 或自带的终端里执行该 `.sh` 脚本。
2. **安装 GitHub CLI (gh)**：
   脚本使用 GitHub 官方命令行工具自动创建发行版。请前往 [GitHub CLI 官网](https://cli.github.com/) 下载并安装。
3. **初始化登录**：
   在终端中运行以下命令并登录您的 GitHub 账号：
   ```bash
   gh auth login
   ```

### 自动发布流程

在 Git Bash 终端运行脚本：

```bash
./release.sh
```

根据终端中的提示：
1. **输入版本号**（例如：`v1.0.0`）。
2. **输入 GitHub 仓库链接**（如果项目尚未关联远程仓库）。
3. **提交与推送选项**：脚本将自动在 `master` 分支提交您开发的所有源码进行本地归档，并让您选择是否将源码分支推送到 GitHub。如果您是公开仓库，建议选择 `n`（不推送 `master` 源码分支）。
4. **编译并自动切换推送**：脚本将自动调用 PyInstaller 打包、切换到 `main` 发布分支、清空源码、配置忽略名单、向远程强制推送到 `main` 分支，最后使用 GitHub CLI 自动发布 Release 并将 EXE 拖拽上传！完成之后，脚本会自动把开发环境切换回 `master` 分支，恢复开发状态。
